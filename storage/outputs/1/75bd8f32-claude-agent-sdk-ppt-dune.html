Act II · 05 / 18
三层积木模型

Level 1 → Level 2 → Level 3

Level 1
原子Skill
系统内置原子技能（WebSearch、FileParser、Weather等）+ MCP协议接入的外部工具
特点：单一功能、无状态、幂等执行、可被复合Skill调用
Foundation
Level 2
复合Skill
YAML DSL编排多个原子Skill，支持DAG并行、条件分支、变量传递
特点：可视化编排、可复用、版本管理、权限控制
Composer
Level 3
业务Agent
挂载复合Skill的智能体，具备多轮对话、上下文记忆、意图识别能力
特点：面向用户、可配置Persona、支持多模型切换
Runtime
设计哲学：像搭积木一样构建AI能力。原子Skill是最小单元，复合Skill是中间层编排，业务Agent是最终交付形态。

name: travel-planner
version: 1.2.0

Act III · 10 / 18
模型接入层设计

双流式架构

Path A

Anthropic原生路径

  • 真·Token-by-Token流式输出
  • 原生Tool Use协议
  • Claude 3.5 Sonnet/Haiku
  • Thinking模式支持
  • 原生多模态（Vision）
最佳体验，推荐用于核心业务场景
Path B

OpenAI兼容路径

  • Function Calling循环适配
  • 支持GPT-4/GPT-3.5
  • 支持DeepSeek-V3/R1
  • 支持Qwen/GLM等国产模型
  • 统一接口，灵活切换
灵活扩展，用于多模型A/B测试
架构优势：统一抽象层屏蔽底层差异，上层业务代码无需关心具体模型，可随时切换或并行调用多个模型。

Quick Start

本地开发

# 1. 克隆仓库
git clone https://github.com/...
cd agent-platform

# 2. 环境配置
cp .env.example .env
# 编辑 .env 配置API Key

# 3. 启动服务
docker-compose up -d

# 4. 访问应用
open http://localhost:3000

生产部署
